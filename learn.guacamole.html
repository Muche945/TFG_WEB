¡Aprende sobre Apache Guacamole!

En este apartado de la web aprendereas sobre el uso de Apache Guacamole

Enfoque principal en la empresa

En esta empresa Apache Guacamole se va a utilizar para la gestión de una conexión a otro equipo que tiene configurado Portainer para la gestión de dockers

Además de que se gestionará el uso de las conexiones y el acceso mediante los roles y los grupos asignados a los usuarios, en este encontraremos 3 roles: administradores, usuarios de la empresa y usuarios externos. Siendo estos últimos a los que va dirigido esta web en la que se explica lo utilizado para la atracción y entendimiento de gente externa a la empresa.

Atributos de Apache Guacamole

Es evidente que el uso de Apache Guacamole está más que estandarizado para la gestión de conexiones a equipos remotos por los siguientes motivos.

  • Fácil escalabilidad, ya que este nos permite tener una alta disponibilidad de usuarios.
  • Podremos acceder desde cualquier navegador siempre que nos encontremos en la misma red de la empresa.
  • Interfaz sencilla y clara lo cual nos permite un fácil uso y administración de esta
  • Está dockerizado, lo que nos facilita el uso de esta a la hora de tener que reiniciar el servicio si fuera necesario.
  • Compatible con todo tipo de protocolos, desde VNC pasando por RDP y SSH.

Funcionalidades

Las principales funcionalidades de Apache Guacamole son la gestión de las conexiones a equipos a través de protocolos como el RDP.
